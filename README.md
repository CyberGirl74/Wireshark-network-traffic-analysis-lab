# Wireshark Network Traffic Analysis Lab

## Overview

This hands-on lab was completed using Wireshark to capture and analyze network traffic on a home Wi-Fi network.

The objective was to gain practical experience identifying common network protocols, monitoring packet activity, and understanding how devices communicate across a network.

---

## Tools Used

- Wireshark
- Windows 11
- Home Wi-Fi Network

---

## Activities Performed

- Captured live network traffic
- Monitored DNS queries and responses
- Investigated mDNS (Multicast DNS) traffic
- Identified TLS encrypted communications
- Observed QUIC protocol traffic
- Examined ARP activity
- Filtered packets using Wireshark display filters

---

## Key Findings

### DNS Traffic

Observed DNS queries used to resolve domain names into IP addresses.

Examples included:

- Google services
- Comcast DNS infrastructure
- Device name resolution

### mDNS Traffic

Observed multicast DNS traffic generated by devices on the local network.

Examples included:

- Smart TV advertisements
- AirPlay service discovery
- Local device announcements

### TLS Traffic

Captured encrypted TLS traffic between local devices and external services.

This demonstrated how modern internet communications are protected through encryption.

### QUIC Traffic

Observed QUIC protocol communications operating over UDP.

QUIC is commonly used by modern web applications to improve performance and reduce latency.

### ARP Traffic

Observed Address Resolution Protocol (ARP) traffic used by devices to identify hardware addresses on the local network.

---

## Lessons Learned

This lab provided valuable experience using Wireshark to analyze real network traffic.

Key takeaways included:

- Devices constantly communicate on a network
- Smart devices generate significant background traffic
- DNS and mDNS play important roles in device discovery
- Most modern web traffic is encrypted
- Wireshark is a powerful tool for network monitoring and troubleshooting
